# 个人品牌网站接口文档

## 1. 内容管理接口

### 1.1 获取文章列表

```
GET /api/articles
```

| 参数名    | 类型      | 必填 | 说明        |
| ------ | ------- | -- | --------- |
| pinned | boolean | 否  | 是否只返回置顶文章 |
| limit  | number  | 否  | 返回条数，默认10 |

响应示例：

```json
[
  {
    "id": "uuid",
    "title": "文章标题",
    "content": "文章内容",
    "pinned": true,
    "created_at": "2026-01-21T00:00:00Z"
  }
]
```

***

### 1.2 更新文章内容（后台）

```
PUT /api/articles/:id
```

| 参数名     | 类型      | 必填 | 说明   |
| ------- | ------- | -- | ---- |
| title   | string  | 是  | 文章标题 |
| content | string  | 是  | 文章内容 |
| pinned  | boolean | 否  | 是否置顶 |

***

### 1.3 获取履历信息

```
GET /api/resume
```

响应示例：

```json
{
  "name": "姓名",
  "title": "职位",
  "bio": "个人简介",
  "experience": [
    {
      "company": "公司名",
      "role": "职位",
      "duration": "时间段"
    }
  ]
}
```

***

### 1.4 更新履历信息（后台）

```
PUT /api/resume
```

| 参数名        | 类型     | 必填 | 说明   |
| ---------- | ------ | -- | ---- |
| name       | string | 是  | 姓名   |
| title      | string | 是  | 职位   |
| bio        | string | 是  | 个人简介 |
| experience | array  | 否  | 工作经历 |

***

## 2. 用户交互接口

### 2.1 提交留言

```
POST /api/contact
```

| 参数名     | 类型     | 必填 | 说明   |
| ------- | ------ | -- | ---- |
| name    | string | 是  | 姓名   |
| email   | string | 是  | 邮箱   |
| message | string | 是  | 留言内容 |

响应示例：

```json
{
  "status": "success",
  "message": "感谢您的留言，我会尽快回复！"
}
```

***

## 3. 系统配置接口

### 3.1 获取特效开关状态

```
GET /api/config/effects
```

响应示例：

```json
{
  "enabled": true
}
```

***

### 3.2 更新特效开关状态（后台）

```
PUT /api/config/effects
```

| 参数名     | 类型      | 必填 | 说明     |
| ------- | ------- | -- | ------ |
| enabled | boolean | 是  | 是否启用特效 |

***

## 4. 图片资源接口

### 4.1 获取图片列表（用于预览）

```
GET /api/images
```

响应示例：

```json
[
  {
    "id": "uuid",
    "url": "https://example.com/image.jpg",
    "alt": "图片描述"
  }
]
```

***

## 5. 错误码说明

| 状态码 | 含义        |
| --- | --------- |
| 200 | 请求成功      |
| 400 | 参数错误      |
| 401 | 未授权（后台接口） |
| 404 | 资源不存在     |
| 500 | 服务器错误     |

***

## 6. 使用说明

* 所有时间格式为 ISO 8601 标准

* 所有接口返回统一格式：`{ status, data, message }`

* 后台接口需携带身份验证令牌（如 JWT）
