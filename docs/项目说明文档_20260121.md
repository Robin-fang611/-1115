# 个人品牌网站项目说明文档

**生成日期**: 2026-01-21

## 1. 项目概览
本项目是一个基于 Next.js 构建的个人品牌展示网站，旨在为个人创作者、独立开发者或设计师提供一个展示作品、分享经历和管理个人品牌的平台。项目内置了轻量级的内容管理系统 (CMS)，允许用户通过可视化后台直接管理网站内容，无需编写代码。

## 2. 技术栈
*   **框架**: [Next.js 16 (App Router)](https://nextjs.org/) - 现代化的 React 框架，支持服务端渲染 (SSR) 和静态生成 (SSG)。
*   **语言**: TypeScript - 提供静态类型检查，增强代码健壮性。
*   **样式**: Tailwind CSS - 原子化 CSS 框架，配合 `clsx` 和 `tailwind-merge` 实现灵活的样式管理。
*   **UI 组件**: Radix UI (基础组件) + Lucide React (图标库)。
*   **状态管理**: Zustand - 轻量级全局状态管理库，用于管理内容数据 (`useContentStore`)。
*   **数据存储**: JSON 文件存储 (`src/data/siteData.json`) - 无需数据库，通过文件读写实现数据持久化，便于部署和迁移。

## 4. 核心功能板块

### 4.1 前台展示
*   **首页 (Home)**: 个性化 Hero 区域，展示 Slogan、动态更新和社交媒体链接。
*   **关于我 (About)**: 展示个人履历、技能标签 (Tags)、时间轴经历 (Timeline) 和个人价值观。
*   **项目进展 (Projects)**: 展示核心业务、商业化解决方案及未来规划 (Roadmap)。
*   **分享与手记 (Share)**: 博客系统，包含“创业分享”和“成长手记”两个板块，支持文章分类、置顶和封面图展示。

### 4.2 后台管理 (/admin)
*   **可视化编辑**: 提供表单界面，直接修改 `siteData.json` 中的内容，所见即所得。
*   **留言管理**: 查看和管理访客通过“联系我”提交的留言信息。
*   **文章发布**: 撰写和发布新的博客文章，支持设置分类和置顶。
*   **全局设置**: 修改网站标题、Logo 文案以及测试邮件服务配置。

## 5. 数据管理机制
本项目采用 **“基于文件的 CMS”** 架构：
1.  **读取**: 应用启动或页面加载时，通过 API 读取 `src/data/siteData.json`。
2.  **更新**: 在后台 (/admin) 修改内容后，前端发送 POST 请求到 `/api/site-data`。
3.  **持久化**: 后端 API 接收数据并重写 `siteData.json` 文件，实现数据更新。
*注意：在本地开发模式下，文件修改会触发热更新；在生产环境中，需确保服务器对 `data` 目录有写入权限。*

## 6. 快速开始
1.  **安装依赖**:
    ```bash
    npm install
    # 或 yarn install
    ```
2.  **启动开发服务器**:
    ```bash
    npm run dev
    ```
3.  **访问网站**:
    *   前台: `http://localhost:3000`
    *   后台: `http://localhost:3000/admin`

## 7. 部署建议
*   推荐使用 **Vercel** 进行部署，本项目已针对 Vercel 进行了优化。
*   由于使用了文件系统作为数据库，在 Serverless 环境（如 Vercel）中，对 `siteData.json` 的修改是临时的（非持久化）。若需在生产环境长期保存后台修改的数据，建议后续接入 MongoDB 或 PostgreSQL，或使用 Vercel Blob/KV 存储。
